Replace the Plan 01 stub `internal/templates/settings.templ` with the full implementation. Follow CLAUDE.md design system rules (glass- prefix, CSS custom properties, Bricolage Grotesque headings, Outfit body).

**SettingsPage(plugins []settings.PluginSettingsViewModel) templ component:**
- Wrap in `@Layout("Settings - First Sip") { ... }`
- Add glass-navbar with "Settings" title and back-to-dashboard link
- Main content area with `.dashboard-content` max-width
- Heading: "Plugin Settings" in `--font-display`
- For each plugin: render `@PluginAccordionRow(plugin)`
- If no plugins: empty state message
- Include `<script>` block at bottom for:
  - Accordion expand/collapse toggle (client-side `.settings-expanded` class toggle on row click)
  - "Saved ✓" revert logic: `htmx.on('htmx:afterSwap', ...)` detects `[data-saved="true"]` button, reverts text after 2s (per research Pattern 6)
  - Auto-expand on enable: listen for `settings-auto-expand` HX-Trigger from toggle response

**PluginAccordionRow(plugin settings.PluginSettingsViewModel) templ component:**
- `id="plugin-row-{pluginID}"` for HTMX targeting
- **Collapsed header** (always visible): `.settings-plugin-row` glass-card-style row
  - Status dot: `<span class="settings-status-dot settings-status-{healthColor}"></span>` (green/yellow/red, gray if no runs)
  - Plugin icon (emoji from plugin.Icon) + plugin name in `--font-display`
  - Enable/disable toggle: `<input type="checkbox" class="settings-toggle-input" hx-post="/api/settings/{pluginID}/toggle" hx-target="#plugin-row-{pluginID}" hx-swap="outerHTML" checked?={plugin.Enabled} />` with `.settings-toggle` label wrapper and `.settings-toggle-slider` span
  - Expand/collapse chevron arrow (rotate on expand via CSS)
  - Click handler on the row (excluding toggle) toggles `.settings-expanded` class

- **Expanded content** (`.settings-plugin-expanded`, hidden by default, shown when `.settings-expanded`):
  - Two sections side by side on desktop, stacked on mobile:
    - Left: Settings form (`@PluginSettingsForm(plugin)`)
    - Right: Plugin status (`@PluginStatusSection(plugin)`)

**PluginSettingsForm(plugin settings.PluginSettingsViewModel) templ component:**
- Wrapped in `<form>` (no native submit — HTMX handles it)
- **Schedule section** (always present): cron expression text input + timezone text input (read-only display for now). Labeled "Schedule" as a subsection.
- **Plugin settings section** (if plugin.HasSchema):
  - For each field in plugin.Fields: render appropriate input based on field.FieldType
  - Each field wrapped in `.settings-field-group`:
    - `<label class="settings-field-label">{field.Label}</label>`
    - Help tooltip icon next to label if field.Description is non-empty: `<span class="settings-tooltip-trigger" title="{description}">?</span>`
    - Input element with `hx-post="/api/settings/{pluginID}/validate-field" hx-trigger="blur changed" hx-target="#error-{field.Key}-{pluginID}" hx-swap="innerHTML"` and hidden `<input type="hidden" name="field_name" value="{field.Key}" />`
    - Error container: `<div id="error-{field.Key}-{pluginID}" class="settings-field-error">{field.Error}</div>`

  **Field rendering by FieldType (Claude's discretion decisions from CONTEXT.md):**
  - `FieldTypeText`: `<input type="text" name="{key}" value="{currentValue}" class="settings-input" />`
  - `FieldTypeInteger`: `<input type="number" name="{key}" value="{currentValue}" class="settings-input" />`
  - `FieldTypeBoolean`: Toggle switch — hidden `<input type="hidden" name="{key}" value="false" />` + `<label class="settings-toggle"><input type="checkbox" name="{key}" value="true" checked?={currentValue == "true"} /><span class="settings-toggle-slider"></span></label>` (handles Pitfall 3)
  - `FieldTypeEnum` (<=4 options): Radio buttons — `for _, opt := range field.EnumValues { <label class="settings-radio-label"><input type="radio" name="{key}" value="{opt}" checked?={opt == currentValue} class="settings-radio" /> {opt}</label> }`
  - `FieldTypeEnum` (>4 options): `<select name="{key}" class="settings-select">` with options
  - `FieldTypeCheckboxGroup`: Checkbox group — `for _, opt := range field.EnumValues { <label class="settings-checkbox-label"><input type="checkbox" name="{key}" value="{opt}" checked?={isChecked(opt, currentValue)} class="settings-checkbox" /> {opt}</label> }` (multiple values submitted for same name — handles Pitfall 4)

- **Save button**: `<button type="button" class="glass-btn glass-btn-primary" hx-post="/api/settings/{pluginID}/save" hx-include="closest form" hx-target="#plugin-row-{pluginID}" hx-swap="outerHTML" if plugin.SaveSuccess { data-saved="true" }>` text: "Saved ✓" if SaveSuccess, else "Save"

**PluginStatusSection(plugin settings.PluginSettingsViewModel) templ component:**
- If plugin.Status is nil: "No runs yet" in muted text
- If present:
  - Last run: formatted time (or "Never")
  - Next run: formatted time (or "No schedule")
  - Recent errors: list of up to 5 errors with timestamp and message in `.settings-error-item` styled containers
  - "Run Now" button: `<button class="glass-btn glass-btn-ghost glass-btn-sm" hx-post="/api/settings/{pluginID}/run-now" hx-swap="none">Run Now</button>`

**Helper templ functions:**
- `isChecked(option string, currentValue string) bool` — for checkbox groups, parse currentValue (comma-separated or check if option is in the array representation)
