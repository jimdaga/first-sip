<decisions>
## Implementation Decisions

### Output JSON contract
- Sidecar wraps CrewAI output as `{summary, sections[]}` — summary for tile display, sections array for structured content
- Section structure: Claude's discretion based on what CrewAI actually outputs
- Summary is generated by the sidecar Python code (not truncation) — extract or generate from full output
- Go side (extractSummary/extractContent) supports new JSON format only — no backwards compatibility with old raw-string format; old runs show as unavailable

### Malformed output handling
- When CrewAI output can't be parsed: Claude's discretion on most robust approach
- When CrewAI completely fails (crash, timeout, empty output): sidecar publishes `{"error": "..."}` error payload to Redis so Go side can mark run as failed
- Dashboard shows generic error message to user: "Briefing generation failed — try again later" (no specific error details)
- Add a manual "Retry" button on failed tiles so user can trigger re-generation

### Timezone
- **Timezone is a USER-level setting, not per-plugin** — remove Timezone field from UserPluginConfig entirely
- Fallback chain: `User.Timezone` → `UTC` (no per-plugin timezone)
- Default when User.Timezone is empty: UTC
- Remove timezone picker from plugin settings UI
- Add timezone picker to user-level settings page
- Whether to notify user about fallback timezone: Claude's discretion
- Timezone resolution timing (schedule-time vs each-fire): Claude's discretion based on existing scheduler patterns

### Dead code cleanup
- Sweep for dead code in files being touched by this phase (not just the 3 listed items)
- Dead code in untouched files: Claude's discretion based on confidence
- Removing per-plugin timezone UI is part of the timezone work, not cleanup
- Listed items: ValidateUserSettings, TileSkeleton, stale scheduler comment

### Claude's Discretion
- Section structure within the `sections[]` array
- Malformed output fallback strategy (wrap raw vs fail)
- Timezone fallback notification in settings UI
- Timezone resolution timing (schedule-time vs each-fire)
- Dead code sweep scope beyond touched files

</decisions>
