   CRITICAL — Embed BOTH collapsed and expanded content in the same TileCard component:

   - **Collapsed content div** (class="tile-collapsed-content"): Visible by default, hidden when .tile-expanded is added.
     - If tile.HasError and tile.LastSuccessfulSummary is not empty: show LastSuccessfulSummary with error icon overlay.
     - Else if tile.BriefingSummary is not empty: show summary in `.tile-summary-text` (3-line clamp).
     - Else if tile.NextRunAt is not nil: show waiting message "Your first briefing is scheduled for {formatted time}".
     - Else: show generic waiting message.

   - **Expanded content div** (class="tile-expanded-content"): Hidden by default, shown when .tile-expanded is added.
     - Close button (X) in top-right: `<button class="tile-close-btn" onclick="collapseTile(this)">` with X SVG icon.
     - If tile.HasError and tile.LastSuccessfulContent is not empty: render LastSuccessfulContent as HTML using `@templ.Raw(tile.LastSuccessfulContent)` with error banner at top.
     - Else if tile.BriefingContent is not empty: render full content as HTML using `@templ.Raw(tile.BriefingContent)`.
     - Else: show the same collapsed content (summary or waiting message).

   This approach means the full briefing content is ALREADY IN THE DOM when the page loads. The JS `expandTile()` function only toggles the CSS class — no additional server request needed for expand. This is the "embed full content as hidden div revealed by JS" approach.

   d. **TileSkeleton** — `templ TileSkeleton(pluginID uint, tileSize string)`:
   - Skeleton tile with data-tile-size, hx-get="/api/tiles/{pluginID}", hx-trigger="load", hx-swap="outerHTML".
   - Contains skeleton-icon div, skeleton-line-title div, two skeleton-line divs, one skeleton-line-short div.

   e. **Onboarding empty state** — `templ TileOnboarding()`:
   - Glass card with class `tile-onboarding`.
   - Text: "Enable your first plugin to get started".
   - Button linking to `/settings` (Phase 12 will create this page): `<a href="/settings" class="glass-btn glass-btn-primary">Browse Plugins</a>`.

   f. **JavaScript for expand/collapse** — add a `<script>` block at the end of DashboardPage (inside the Layout children):
   ```javascript
   function expandTile(el) {
     // Don't expand if clicking close button
     if (el.classList.contains('tile-expanded')) return;
     // Collapse any currently expanded tile
     var expanded = document.querySelector('.tile-expanded');
     if (expanded) expanded.classList.remove('tile-expanded');
     // Expand this tile — CSS toggles visibility of collapsed/expanded content divs
     el.classList.add('tile-expanded');
   }
   function collapseTile(btn) {
     var tile = btn.closest('.tile-card');
     if (tile) tile.classList.remove('tile-expanded');
     event.stopPropagation();
   }
   // Click outside to collapse
   document.addEventListener('click', function(e) {
     var expanded = document.querySelector('.tile-expanded');
     if (expanded && !expanded.contains(e.target)) {
       expanded.classList.remove('tile-expanded');
     }
   });
   // SortableJS initialization (re-runs on HTMX swap)
   htmx.onLoad(function(content) {
     var sortables = content.querySelectorAll('.sortable');
     for (var i = 0; i < sortables.length; i++) {
       var sortable = sortables[i];
       var instance = new Sortable(sortable, {
         animation: 150,
         ghostClass: 'sortable-ghost',
         onEnd: function(evt) {
           this.option('disabled', true);
         }
       });
       sortable.addEventListener('htmx:afterRequest', function() {
         instance.option('disabled', false);
       });
     }
   });
   ```

   NOTE: The expand/collapse is purely client-side CSS class toggling. The `expandTile()` function adds `.tile-expanded` to the tile-card div, which triggers CSS rules that hide `.tile-collapsed-content` and show `.tile-expanded-content`. No server round-trip is needed because the full briefing content (from `BriefingContent` field) is already embedded in the hidden expanded div.
