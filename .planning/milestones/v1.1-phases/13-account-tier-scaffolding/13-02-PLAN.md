  <files>

  <action>
    1. Extend `internal/settingsvm/settingsvm.go`:
       - Add `TierInfo` struct:
         ```go
         type TierInfo struct {
             TierName          string // "free" or "pro"
             MaxEnabledPlugins int    // -1 = unlimited
             EnabledCount      int    // currently enabled count
             AtPluginLimit     bool   // EnabledCount >= MaxEnabledPlugins (and not unlimited)
             MinFrequencyHours int    // minimum hours between runs for this tier
             UpgradeURL        string // "/pro"
         }
         ```
       - Add `SettingsPageViewModel` struct wrapping plugins + tier:
         ```go
         type SettingsPageViewModel struct {
             Plugins  []PluginSettingsViewModel
             TierInfo TierInfo
         }
         ```
       - Add `IsDisabledByTier bool` field to `PluginSettingsViewModel` — set true for non-enabled plugins when at limit
       - Add `FrequencyError string` field to `PluginSettingsViewModel` — set when save is rejected for frequency violation

    2. Extend `internal/settings/viewmodel.go`:
       - Add a `BuildTierInfo(db *gorm.DB, tierService *tiers.TierService, userID uint) (TierInfo, error)` function:
         - Calls `tierService.GetUserTier(userID)` to get tier
         - Calls `tierService.GetEnabledCount(userID)` for current count
         - Computes AtPluginLimit: count >= MaxEnabledPlugins AND MaxEnabledPlugins >= 0
         - Returns TierInfo with UpgradeURL set to "/pro"
       - Modify `BuildPluginSettingsViewModels` to accept a `TierInfo` parameter
       - After building all viewmodels, if `tierInfo.AtPluginLimit`, iterate and set `IsDisabledByTier = true` on all viewmodels where `Enabled == false`

    3. Modify handler signatures in `internal/settings/handlers.go`:
       - `SettingsPageHandler(db *gorm.DB, pluginDir string, tierService *tiers.TierService)`:
         - Build TierInfo via BuildTierInfo
         - Build plugins viewmodel (pass tierInfo for disabled state)
         - Construct SettingsPageViewModel{Plugins: plugins, TierInfo: tierInfo}
         - Change `templates.SettingsPage()` call to pass SettingsPageViewModel instead of []PluginSettingsViewModel

       - `TogglePluginHandler(db *gorm.DB, pluginDir string, tierService *tiers.TierService)`:
         - Determine newState (the target enabled state after toggle)
         - If newState is true (enabling), call `tierService.CanEnablePlugin(user.ID)`
         - If canEnable is false: re-build the single plugin viewmodel with IsDisabledByTier=true and re-render the accordion row. Use an HTMX OOB swap to also update the counter. Return early (do NOT toggle in DB).
         - If enabling succeeds OR disabling: proceed with existing toggle logic
         - After toggle completes, re-build TierInfo and include an OOB swap of the counter fragment in the response alongside the accordion row

       - `SaveSettingsHandler(db *gorm.DB, pluginDir string, tierService *tiers.TierService)`:
         - Before existing schema validation, check frequency: if cronExpr is present, call `tierService.CanUseFrequency(user.ID, cronExpr)`
         - If frequency not allowed: set FrequencyError on the viewmodel ("Schedules faster than once daily require Pro. Your tier allows minimum {MinFrequencyHours}h intervals.") and re-render with error, return early
         - Otherwise proceed with existing save logic

    4. Update `cmd/server/main.go`:
       - Import `internal/tiers`
       - Construct: `tierService := tiers.New(db)`
       - Pass tierService to all three handler constructors:
         ```go
         protected.GET("/settings", settings.SettingsPageHandler(db, cfg.PluginDir, tierService))
         protected.POST("/api/settings/:pluginID/toggle", settings.TogglePluginHandler(db, cfg.PluginDir, tierService))
         protected.POST("/api/settings/:pluginID/save", settings.SaveSettingsHandler(db, cfg.PluginDir, tierService))
         ```
       - Add GET route for /pro page: `protected.GET("/pro", func(c *gin.Context) { render(c, templates.ProComingSoonPage()) })`
       - Add POST route for email capture: `protected.POST("/api/pro/notify", proNotifyHandler())` — handler logs the email with slog.Info and returns a "thank you" HTML fragment. No DB table needed for scaffolding.
  </action>
