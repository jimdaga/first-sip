           updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
           deleted_at TIMESTAMPTZ,
           name VARCHAR(50) NOT NULL UNIQUE,

    1. Create `internal/tiers/service.go` with `TierService` struct:
       - Constructor: `func New(db *gorm.DB) *TierService`

       - `GetUserTier(userID uint) (*models.AccountTier, error)`:
         - Query user's AccountTierID from users table (SELECT account_tier_id WHERE id = ?)
         - If AccountTierID is nil, fall back to looking up `name = 'free'` in account_tiers
         - Return the tier struct

       - `CanEnablePlugin(userID uint) (bool, *models.AccountTier, error)`:
         - Call GetUserTier to get the user's tier
         - If MaxEnabledPlugins < 0 (unlimited), return true
         - Count currently enabled plugins: `db.Model(&plugins.UserPluginConfig{}).Where("user_id = ? AND enabled = true AND deleted_at IS NULL", userID).Count(&count)`
         - Return count < MaxEnabledPlugins, tier, nil

       - `CanUseFrequency(userID uint, cronExpr string) (bool, *models.AccountTier, error)`:
         - Call GetUserTier
         - If MinFrequencyHours <= 0, return true (no restriction)
         - Parse cronExpr using `cron.NewParser(cron.Minute | cron.Hour | cron.Dom | cron.Month | cron.Dow)`
         - Compute interval: `schedule.Next(now)` then `schedule.Next(next1)`, interval = next2.Sub(next1)
         - Convert tier.MinFrequencyHours to duration: `time.Duration(tier.MinFrequencyHours) * time.Hour`
         - Return interval >= minDuration, tier, nil
         - Handle parse errors by returning false with error

       - `GetEnabledCount(userID uint) (int, error)`:
         - Simple count query for tier UI counter display
         - Returns the count of enabled plugins for the user

       Import robfig/cron/v3 (already in go.mod). Import internal/models and internal/plugins.

       - Set `AccountTierID: &freeTier.ID` on the new User struct before `db.Create(&user)`
       - If the free tier lookup fails (should not happen after seeding), log a warning and continue without assigning a tier — the TierService GetUserTier NULL fallback handles this gracefully
