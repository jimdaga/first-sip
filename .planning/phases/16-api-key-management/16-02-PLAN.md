  <name>Task 1: Create view model, handlers, and register routes</name>
  <files>
    internal/apikeys/viewmodel.go
    internal/apikeys/handlers.go
    cmd/server/main.go
  </files>
  <action>
    1. Create `internal/apikeys/viewmodel.go`:
       - Package `apikeys`.
       - Define `APIKeyViewModel` struct: `ID uint`, `KeyType string`, `Provider string`, `ProviderName string`, `MaskedValue string`.
       - Define `APIKeysPageViewModel` struct: `LLMKeys []APIKeyViewModel`, `TavilyKey *APIKeyViewModel` (nil if not set), `Providers []LLMProvider`, `PreferredProvider string`, `PreferredModel string`.
       - Define `func BuildViewModel(db *gorm.DB, user *models.User) APIKeysPageViewModel`:
         - Call `GetKeysForUser(db, user.ID)` to get all keys.
         - Separate into LLM keys (key_type="llm") and Tavily key (key_type="tavily").
         - For each key, create `APIKeyViewModel` with `MaskedValue: MaskAPIKey(key.EncryptedValue)`.
         - Map provider IDs to display names using `GetProviderByID`.
         - Set `PreferredProvider: user.LLMPreferredProvider`, `PreferredModel: user.LLMPreferredModel`.
         - Set `Providers: SupportedLLMProviders`.

    2. Create `internal/apikeys/handlers.go`:
       - Package `apikeys`.
       - Duplicate `render` and `getAuthUser` helpers from `internal/settings/handlers.go` (established pattern to avoid import cycles).
       - `func PageHandler(db *gorm.DB) gin.HandlerFunc`:
         - Get authenticated user via `getAuthUser`.
         - Build view model via `BuildViewModel(db, user)`.
         - Get sidebar plugins via `dashboard.GetSidebarPlugins(db, user.ID)`.
         - Render `templates.APIKeysSettingsPage(vm, sidebarPlugins)`.
       - `func SaveKeyHandler(db *gorm.DB) gin.HandlerFunc`:
         - Get auth user. Parse form values: `key_type` ("llm" or "tavily"), `provider` (for LLM keys), `api_key` (the plaintext value).
         - Validate: `api_key` must not be empty. For LLM keys, `provider` must be in SupportedLLMProviders.
         - Call `SaveKey(db, user.ID, keyType, provider, apiKey)`.
         - On success: rebuild view model, render the `#api-keys-section` fragment via `templates.APIKeysSection(vm)`.
         - On error: render an error alert fragment.
       - `func DeleteKeyHandler(db *gorm.DB) gin.HandlerFunc`:
         - Get auth user. Parse `:id` URL param.
         - Call `DeleteKey(db, user.ID, keyID)`.
         - Rebuild view model, render `#api-keys-section` fragment.
       - `func SaveLLMPreferenceHandler(db *gorm.DB) gin.HandlerFunc`:
         - Get auth user. Parse form values: `provider`, `model`.
         - Call `SaveLLMPreference(db, user.ID, provider, model)`.
         - Rebuild view model, render preference section fragment.

    3. Register routes in `cmd/server/main.go`:
       - Add import: `"github.com/jimdaga/first-sip/internal/apikeys"`.
       - In the protected group, add:
         ```go
         protected.GET("/settings/api-keys", apikeys.PageHandler(db))
         protected.POST("/api/user/api-keys", apikeys.SaveKeyHandler(db))
         protected.POST("/api/user/api-keys/:id/delete", apikeys.DeleteKeyHandler(db))
         protected.POST("/api/user/llm-preference", apikeys.SaveLLMPreferenceHandler(db))
         ```
       - Place these AFTER the existing settings routes block for readability.
  </action>
  <verify>
    Run `make build` — compiles without errors (confirms handler and route registration are valid).
  </verify>
  <done>
    View model builder transforms decrypted keys into masked display models. Four handlers exist: page render, save key (upsert), delete key (soft delete), save LLM preference. Routes are registered in main.go's protected group. All handlers follow the settings/handlers.go pattern with duplicated render/getAuthUser helpers.
  </done>
