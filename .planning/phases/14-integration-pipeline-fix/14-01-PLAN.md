**Sidecar (sidecar/executor.py):**

Add three methods to `PluginExecutor`:

1. `_wrap_output(self, raw: str) -> str` — calls `_extract_summary` and `_build_sections`, returns `json.dumps({"summary": summary, "sections": sections})`.

2. `_extract_summary(self, raw: str) -> str` — split on `\n\n`, skip lines starting with `#`, return first non-heading paragraph truncated to 300 chars. If no non-heading paragraph found, return `raw[:300]`.

3. `_build_sections(self, raw: str) -> list` — split on `^##\s+` (multiline regex). If only one part (no `##` headings), return `[{"title": "Briefing", "content": raw.strip()}]`. Otherwise, for each part after the first, split on first `\n` to get title and content. Return list of `{"title": str, "content": str}`.

Update `execute()` method: after getting `raw_output = result.raw if hasattr(result, 'raw') else str(result)`, call `structured = self._wrap_output(raw_output)` and pass `output=structured` to `PluginResult`. This produces a valid JSON string.

For malformed/empty output fallback: if `raw_output` is empty or None, wrap as `json.dumps({"summary": "Briefing generated", "sections": [{"title": "Content", "content": ""}]})` rather than failing.

Do NOT change `PluginResult.output` type in `sidecar/models.py` — it stays `str | None`. The JSON string is correct (Pitfall 1 from research).

**Go — viewmodel (internal/dashboard/viewmodel.go):**

Add `OutputSection` struct with `Title string json:"title"` and `Content string json:"content"`.

Update `PluginRunOutput` struct to add `Sections []OutputSection json:"sections"` field. Keep existing `Content string json:"content"` for legacy fallback.

Update `extractContent(output []byte) string`:
- If `json.Unmarshal` fails, return `""` (old malformed runs gracefully show empty).
- If `out.Sections` has items, build HTML: for each section, write `<h3>{escaped title}</h3>` (skip if title is empty or "Briefing") then `<p>{content}</p>`. Use `html/template` or `templ.EscapeString` for title escaping. Return concatenated HTML string.
- Otherwise fall back to `out.Content` (legacy).

Leave `extractSummary` as-is — it already returns `out.Summary` with fallback.

**Go — stream handler (internal/streams/handler.go):**

Add a `json.Valid([]byte(result.Output))` check before `datatypes.JSON(result.Output)`. If invalid, log a warning with slog and mark the PluginRun as failed (set Status to "failed") instead of storing garbage. This is a safety net — after the sidecar fix, all output should be valid JSON.
