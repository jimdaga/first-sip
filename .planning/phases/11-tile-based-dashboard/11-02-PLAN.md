<task type="auto">
  <name>Task 1: Create dashboard view model and query layer</name>
  <files>internal/dashboard/viewmodel.go</files>
  <action>
Create `internal/dashboard/viewmodel.go` in a new `dashboard` package.

1. Define `TileViewModel` struct:
   ```go
   type TileViewModel struct {
       PluginID     uint
       PluginName   string
       PluginIcon   string   // emoji from plugin YAML
       TileSize     string   // "1x1", "2x1", "2x2"
       DisplayOrder *int
       Enabled      bool

       // Latest run data (zero values if no runs yet)
       LatestRunStatus string   // pending/processing/completed/failed or ""
       LatestRunAt     *time.Time
       NextRunAt       *time.Time
       BriefingSummary string   // 2-3 line summary from PluginRun.Output JSON
       HasContent      bool     // true if a completed run exists

       // Error overlay: show last successful content even if latest run failed
       LastSuccessfulSummary string
       HasError              bool   // true if latest run is failed
   }
   ```

2. Define `PluginRunOutput` struct for parsing JSONB output:
   ```go
   type PluginRunOutput struct {
       Summary string `json:"summary"`
       Content string `json:"content"`
   }
   ```

3. Create `getDashboardTiles(db *gorm.DB, userID uint) ([]TileViewModel, error)`:
   - Step 1: Raw SQL query for enabled user plugin configs joined with plugins table, ordered by `display_order ASC NULLS LAST, p.name ASC`. Select: plugin_id, plugin name, icon, tile_size, display_order, cron_expression, timezone.
   - Step 2: Raw SQL `DISTINCT ON (plugin_id)` query for latest plugin runs for this user. ORDER BY must be `plugin_id, created_at DESC`. Select: plugin_id, status, output, error_message, completed_at, created_at.
   - Step 3: Build a `map[uint]LatestRun` from step 2 results for O(1) lookup.
   - Step 4: For each config row, build TileViewModel. If latest run exists, populate status/timing/summary. If status is "failed", also query for last successful run (status=completed) for that plugin — or include it in the DISTINCT ON by doing a second query specifically for completed runs.
   - Step 5: Compute NextRunAt using `computeNextRun(cronExpr, timezone string) *time.Time`.
   - Return sorted slice of TileViewModels.

4. Create `computeNextRun(cronExpr, timezone string) *time.Time`:
   - Parse cron expression using `robfig/cron/v3` parser (same parser pattern as in models.go — import and use `cronParser` or create a local one with same options).
   - Load timezone with `time.LoadLocation(timezone)`, fallback to UTC on error.
   - Get `time.Now().In(loc)` and call `schedule.Next(now)`.
   - Return pointer to the next time, or nil if cron expression is empty/invalid.

5. Create `extractSummary(output []byte) string`:
   - Unmarshal output JSON into PluginRunOutput. Return `.Summary` field.
   - If unmarshal fails or summary is empty, try to extract first 200 chars of `.Content` as fallback.
   - Return empty string if output is nil or empty.

6. Create `timeAwareGreeting(name, timezone string) string`:
   - Load timezone location (fallback UTC).
   - Get current hour in user's timezone.
   - Return "Good morning, {name}" (before 12), "Good afternoon, {name}" (12-17), "Good evening, {name}" (17+).

7. Create `formatTimingTooltip(tile TileViewModel) string`:
   - Format: "Last run: {relative time} \u00b7 Next: {relative time}" (using relative time like "2 hours ago", "in 30 minutes").
   - If no last run: "No runs yet \u00b7 Next: {time}".
   - If no next run: "Last run: {time} \u00b7 No schedule".
   - Helper: `formatRelativeTime(t time.Time) string` — compute difference from now, return human-readable string.

8. Create `GetSingleTile(db *gorm.DB, userID, pluginID uint) (*TileViewModel, error)`:
   - Used by the per-tile HTMX polling endpoint. Queries single plugin config + latest run for one plugin.
   - Same logic as getDashboardTiles but filtered to one plugin_id.
  </action>
