  <action>
Create `internal/settings/viewmodel.go` package `settings` with:

**FieldType constants and FieldViewModel struct** (per research Pattern 1):
- `FieldType` string type with constants: `FieldTypeText`, `FieldTypeEnum`, `FieldTypeInteger`, `FieldTypeBoolean`, `FieldTypeCheckboxGroup`
- `FieldViewModel` struct: Key, Label, Description, FieldType, Required, Default string, EnumValues []string, CurrentValue string, Error string

**PluginSettingsViewModel** — the top-level view model for each plugin accordion row:
- PluginID uint, PluginName string, Description string, Icon string, Enabled bool
- Fields []FieldViewModel (nil if plugin has no schema)
- HasSchema bool
- HasRequiredFields bool (for auto-expand on enable)
- Status *PluginStatusViewModel (nil if no runs yet)
- CronExpression string, Timezone string (for schedule section — editable in form)
- SaveSuccess bool (set to true on successful save — drives "Saved ✓" in template)

**PluginStatusViewModel** struct:
- LastRunAt *time.Time, NextRunAt *time.Time
- RecentErrors []ErrorEntry (max 5)
- HealthColor string ("green", "yellow", "red")

**ErrorEntry** struct: OccurredAt time.Time, Message string

**Key functions:**

1. `schemaToFields(schema *jsonschema.Schema, savedSettings map[string]any, submittedValues map[string]string, fieldErrors map[string]string) []FieldViewModel`
   - Iterate `*schema.Properties` with sorted keys (deterministic order — Pitfall 6)
   - Build required set from `schema.Required`
   - For each property: determine FieldType from `propSchema.Type[0]` (guard nil Type), check `propSchema.Enum` and `propSchema.Items.Enum`
   - Label: use `propSchema.Title` (dereference *string) if non-empty, else humanize key (replace _ with space, title case). Per Claude's discretion from CONTEXT.md.
   - Description: `propSchema.Description` (dereference *string)
   - Default: format `propSchema.Default` as string
   - Value priority: submittedValues > savedSettings > default (SET-05)
   - Enum values: from `propSchema.Enum` for string/integer enums, from `propSchema.Items.Enum` for array checkbox groups
   - Error: look up `fieldErrors["/"+key]` (JSON pointer path from DetailedErrors)

2. `coerceFormValues(rawForm url.Values, schema *jsonschema.Schema) (map[string]any, error)` (per research Pattern 2):
   - For each property in schema: get raw form values by key
   - Switch on `propSchema.Type[0]`: "integer" → `strconv.Atoi`, "boolean" → accept "true"/"on"/"1" as true (absent = false, handles Pitfall 3), "array" → `rawForm[key]` ([]string, handles Pitfall 4), default → `rawVals[0]`
   - Return typed map ready for `schema.Validate()`

3. `validateAndGetFieldErrors(schema *jsonschema.Schema, typedValues map[string]any) map[string]string` (per research Pattern 3):
   - Call `schema.Validate(typedValues)`
   - If valid, return nil
   - Call `result.DetailedErrors()` — returns map keyed by JSON pointer paths like "/frequency"
   - Return as-is (FieldViewModel Error lookup uses "/"+key to match)

4. `validateSingleField(schema *jsonschema.Schema, fieldKey string, rawValue string) string` — for blur validation:
   - Build a minimal map with just this one field, coerced to correct type
   - Validate the full map (schema validates whole object)
   - Extract error for this specific field path "/"+fieldKey, return error message or ""

5. `loadPluginSchema(pluginDir, pluginName, schemaRelPath string) (*jsonschema.Schema, error)` (per research Pattern 7):
   - Return nil, nil if schemaRelPath is empty
   - Build full path: `filepath.Join(pluginDir, pluginName, schemaRelPath)`
   - Read file, compile with `jsonschema.NewCompiler()`, return schema

6. `BuildPluginSettingsViewModels(db *gorm.DB, userID uint, pluginDir string) ([]PluginSettingsViewModel, error)`:
   - Query ALL plugins (not just enabled): `SELECT p.*, upc.enabled, upc.settings, upc.cron_expression, upc.timezone FROM plugins p LEFT JOIN user_plugin_configs upc ON p.id = upc.plugin_id AND upc.user_id = ? AND upc.deleted_at IS NULL WHERE p.deleted_at IS NULL ORDER BY p.name`
   - For each plugin: load schema, convert to FieldViewModel with saved settings, compute status
   - Return sorted list of PluginSettingsViewModel

7. `getPluginStatus(db *gorm.DB, userID, pluginID uint, cronExpr, timezone string) *PluginStatusViewModel`:
   - Query latest run: `ORDER BY created_at DESC LIMIT 1`
   - Query recent failed runs: `WHERE status = 'failed' ORDER BY created_at DESC LIMIT 5`
   - Compute health color: red if latest failed, yellow if any recent failures, green otherwise
   - Compute next run using duplicated `computeNextRun` (same logic as dashboard/viewmodel.go — duplicate per research recommendation, avoid import cycle)
   - Return nil if no runs at all

**Import kaptinlin/jsonschema** — already in go.mod, no new dependency.

**IMPORTANT:** Use `encoding/json` to unmarshal `upc.Settings` (JSONB stored as `datatypes.JSON`) into `map[string]any` for passing to schemaToFields.

**TogglePluginHandler(db *gorm.DB, pluginDir string) gin.HandlerFunc** — POST /api/settings/:pluginID/toggle:
- Get auth user, parse pluginID from URL param
- Find or create UserPluginConfig for this user+plugin (use GORM `FirstOrCreate` with user_id and plugin_id)
- Toggle `enabled` field (flip current value)
- Save to DB
- If enabling and plugin has required settings fields (load schema, check schema.Required), set response header `HX-Trigger: settings-auto-expand` so Plan 02 template can auto-expand
- Re-build this single plugin's PluginSettingsViewModel
- Render the updated accordion row fragment: `templates.PluginAccordionRow(vm)` — stub this in settings.templ too

**SaveSettingsHandler(db *gorm.DB, pluginDir string) gin.HandlerFunc** — POST /api/settings/:pluginID/save:
- Get auth user, parse pluginID
- Load plugin from DB, load schema
- Parse raw form values from `c.Request.PostForm`
- Also extract `cron_expression` and `timezone` from form (separate from plugin-specific settings)
- Validate cron expression if non-empty using `plugins.ValidateCronExpression()`
- Call `coerceFormValues(rawForm, schema)` for plugin-specific settings (exclude cron_expression and timezone from this)
- Call `validateAndGetFieldErrors(schema, typedValues)`
- If errors: rebuild PluginSettingsViewModel with submittedValues (raw form strings) and fieldErrors, render expanded form with errors (SET-05)
- If valid: update UserPluginConfig.Settings (JSON marshal typedValues), update CronExpression and Timezone, save
- On success: rebuild PluginSettingsViewModel with SaveSuccess=true, render expanded form showing "Saved ✓"

**ValidateFieldHandler(db *gorm.DB, pluginDir string) gin.HandlerFunc** — POST /api/settings/:pluginID/validate-field:
- Get auth user, parse pluginID
- Get field name from form param `field_name`
- Get field value from form param matching the field name
- Load schema, call `validateSingleField(schema, fieldName, fieldValue)`
- Return HTML fragment: `<span class="settings-field-error">{error}</span>` or empty string
- Use `c.Header("Content-Type", "text/html")` + `c.String(200, html)` for inline response

**RunNowHandler(db *gorm.DB) gin.HandlerFunc** — POST /api/settings/:pluginID/run-now:
- Get auth user, parse pluginID
- Load UserPluginConfig from DB (must exist and be enabled)
- Load plugin name from plugins table
- Unmarshal settings from UserPluginConfig.Settings into map[string]interface{}
- Call `worker.EnqueueExecutePlugin(pluginID, user.ID, pluginName, settings)`
- Return `c.String(200, "Run triggered")` — simple text response for HTMX
