  <action>
In `internal/worker/worker.go`, modify the `handleExecutePlugin` function to inject user API keys and LLM preferences into the settings map before publishing to Redis Streams.

**Add import** for `"github.com/jimdaga/first-sip/internal/apikeys"`.

**Add a helper function** `findAPIKey(keys []models.UserAPIKey, keyType, provider string) *models.UserAPIKey` that iterates keys and returns the first match on KeyType and Provider, or nil.

**In `handleExecutePlugin`**, after the payload is unmarshaled and before `pluginRunID` is generated, add the key injection block:

1. Fetch user record: `var user models.User; db.WithContext(ctx).First(&user, payload.UserID)`. If this fails, log a warning (do NOT log the error as fatal — degrade gracefully and continue without keys).

2. Fetch API keys: `keys, err := apikeys.GetKeysForUser(db, payload.UserID)`. The AfterFind hook on UserAPIKey automatically decrypts. If this fails, log a warning and continue.

3. Find the LLM key: use `findAPIKey(keys, "llm", user.LLMPreferredProvider)`. If found:
   - Set `payload.Settings["_llm_api_key"]` to `llmKey.EncryptedValue` (already decrypted by AfterFind hook).
   - Check if `payload.Settings["_llm_model"]` already has a value (per-plugin override from LLM-03). If NOT present or empty string, set it to `user.LLMPreferredProvider + "/" + user.LLMPreferredModel`.

4. Find the Tavily key: use `findAPIKey(keys, "tavily", "tavily")`. Note: for Tavily keys, the Provider field is "tavily". If found, set `payload.Settings["_tavily_api_key"]` to `tavilyKey.EncryptedValue`.

5. Ensure `payload.Settings` is not nil before injecting (initialize to `make(map[string]interface{})` if nil).

**CRITICAL: Do NOT log** the values of `_llm_api_key` or `_tavily_api_key`. The existing `logger.Info("Plugin request published to stream", ...)` line must NOT include settings. The `logger.Info("Processing plugin:execute task", ...)` line already does not log settings — keep it that way.

**Do NOT change** the `PluginRequest` struct in `streams/types.go` — the credentials travel inside the existing `Settings map[string]interface{}` field. No schema changes needed.
