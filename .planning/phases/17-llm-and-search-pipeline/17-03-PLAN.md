Add a new property `_llm_model` to the `properties` object with:
- `type`: `"string"`
- `enum`: An array containing `""` (empty string = use account default) followed by all supported provider/model combos in LiteLLM format. The values must match what the worker expects (see `internal/apikeys/providers.go` for the provider IDs and model names):

**Why this works with zero code changes:**
1. The settings UI is schema-driven — `schemaToFields` in `viewmodel.go` will create a `FieldViewModel` with `FieldType = FieldTypeEnum` (because enum is present). Since 11 enum values > 4, `renderField` in `settings.templ` will render it as a `<select>` dropdown.
2. `SaveSettingsHandler` in `handlers.go` iterates `schema.Properties` to build `submittedValues`, then `coerceFormValues` coerces the value to a string. The marshaled JSON includes `_llm_model` in `UserPluginConfig.Settings`.
3. When the worker loads settings from DB and unmarshals them, `_llm_model` is present in the map. The worker's existing guard at line 258 (`if existing, ok := payload.Settings["_llm_model"]; !ok || existing == ""`) correctly:
   - Uses the override when the user selected a specific model (e.g., `"openai/gpt-4o"`)
   - Falls back to the account default when the user left it as `""` (empty string)
