## Cross-Phase Integration

### Wiring Status

| Connection | Status | Details |
|------------|--------|---------|
| Phase 8 → 9: Plugin registry → execution pipeline | Connected | InitPlugins syncs to DB; handleExecutePlugin reads plugin data from task payload |
| Phase 9 → 10: CrewAI execution → scheduling | Connected | EnqueueExecutePlugin called by per-minute scheduler |
| Phase 10 → 11: Scheduling → dashboard tiles | Connected | getDashboardTiles queries plugin_runs created by scheduler-triggered execution |
| Phase 11 → 12: Dashboard → settings UI | Connected | Navigation link, shared UserPluginConfig model, RunNow handler |

### Integration Breaks

**P0: CrewAI output format → JSONB storage**
- `sidecar/executor.py:55` produces raw Markdown string
- `internal/streams/handler.go:35` passes directly to `datatypes.JSON()`
- PostgreSQL JSONB column rejects non-JSON content
- `internal/dashboard/viewmodel.go:320` expects `{"summary": "...", "content": "..."}`
- **Affects:** CREW-05, TILE-02, TILE-04

**P1: Browser timezone → scheduler fallback**
- `POST /api/user/timezone` stores detected timezone in `users.timezone`
- `handlePerMinuteScheduler` only reads `UserPluginConfig.Timezone`, falls back to UTC
- `User.Timezone` is never consulted for scheduling
- **Affects:** SCHED-04 (partial)
